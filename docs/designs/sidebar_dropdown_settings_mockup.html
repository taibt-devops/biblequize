📝 Settings page — Theo SPEC mục 13.2

7 sections (theo thứ tự thông tin):

  • Tài khoản — Tên hiển thị (editable) + Email (readonly với badge xác thực)
  • Ngôn ngữ — UI language + Quiz language tách biệt (theo SPEC 10.2)
  • Âm thanh & Rung — Toggle SFX, slider volume, toggle haptics, toggle reduce-motion
  • Thông báo — 5 toggles individual + giờ yên lặng (theo SPEC 12.3)
  • Quyền riêng tư — Profile visibility (3 levels) + leaderboard toggle + challenge toggle
  • Về BibleQuiz — Version + build + 4 legal/support links
  • Khu vực nguy hiểm — Delete account với warning rõ + button danger style (theo SPEC 14.3, Apple bắt buộc)

Pattern UI thống nhất:

  • Mỗi section là 1 card glass với header (icon + title + desc)
  • Setting row pattern: [label + desc] | [control]
  • Controls: Toggle (boolean) · Segmented (2-3 options) · Select (3+ options) · Slider (range) · Time input · Text input · Button

API endpoints (mới hoặc reuse):

  • PATCH /api/me — update name (existing)
  • PATCH /api/me/settings — bulk update settings (mentioned in SPEC mục 17.3)
  • DELETE /api/me/account — delete account (existing per SPEC 14.3)
  • Settings persist trong DB table user_settings (cần tạo nếu chưa có)

Câu hỏi cho Bui:

  • "Trợ giúp" trong dropdown — link tới /help page riêng (FAQ + contact form), hay external link Facebook group? Hiện chưa có /help page trong code.
  • Settings persistence — backend hiện có endpoint PATCH /api/me/settings chưa? Nếu chưa, cần build cùng với UI.
  • "Xóa tài khoản" — UI confirm modal "nhập XÓA TÀI KHOẢN" có nên vẽ riêng không? (Theo SPEC 14.3 cần modal này.)
  • Avatar upload — không thấy trong SPEC 13.2 nhưng common pattern. Defer hay add row vào Tài khoản section?
