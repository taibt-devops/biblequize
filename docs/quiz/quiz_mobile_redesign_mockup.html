📝 Phân tích bug + giải pháp
🔴 Root cause của bug ở phone đầu tiên:
  • Container câu hỏi có height fixed (probably ~230px) + overflow: hidden → text dài hơn bị cắt cả trên và dưới khi vertical-align center.
  • Font câu hỏi quá lớn (~28-30px) cho mobile width 380-414px — chỉ fit ~6 từ/dòng, câu 130 ký tự cần 6+ dòng.
  • Top area lãng phí — 4 row metadata riêng (counter+book / energy / timer / combo+năng-lượng-bars) ăn ~180px.
  • Còn lại sau metadata + 2 đáp án visible = không đủ chỗ cho question full.

3 thay đổi structural trong redesign:

  • 1. Compact top bar — 1 row duy nhất: [X close] + Câu 7/20 + sách + Timer ring (44px). Tiết kiệm ~80px so với 4-row hiện tại.
  • 2. HUD strip 1 row — 3 pills consolidated: Năng lượng (5 bars mini) · Combo · Điểm. Thay cho 2 row tách biệt + label "COMBO CHUỖI" thừa.
  • 3. Question card flexible: flex: 1 với min-height: 120px, font adaptive theo length:
    • Câu < 80 ký tự → font 21px, font-weight 700, center align
    • Câu 80–180 ký tự → font 18px, font-weight 600, center align
    • Câu > 180 ký tự → font 15px, font-weight 600, left align (dễ đọc hơn cho nhiều dòng)
    • Container có overflow-y: auto với scrollbar mảnh + fade gradient bottom làm hint nếu vẫn cần scroll
✅ Fix CSS chi tiết cho implementation: 
.question-area {
  flex: 1;
  display: flex; flex-direction: column;
  min-height: 0;  /* CRITICAL: cho phép shrink */
  overflow: hidden;
}
.question-card {
  flex: 1;
  min-height: 120px;
  overflow-y: auto;  /* fallback nếu vẫn dài */
  display: flex; align-items: center; justify-content: center;
}
/* JavaScript add class theo length */
const len = question.text.length;
const cls = len < 80 ? 'short' : len < 180 ? 'medium' : 'long';
<div className={`question-text ${cls}`}>

Cải thiện thêm trong redesign:

  • Verse reference hiện rõ dưới câu hỏi (Sáng Thế Ký 37:9) — giúp user học context, không phải chỉ memorize.
  • Timer là circular ring tích hợp với button, không tách thành cell riêng. Khi <5s → màu đỏ + pulse.
  • Answer color mapping A=Coral, B=Sky, C=Gold, D=Sage (per memory) — giữ nguyên, đã đúng.
  • Compact answers cho câu dài: min-height 44px (vs 56px default), padding nhỏ hơn — giải phóng thêm space cho question.

Câu hỏi cho Bui:

  • Approach JS-based length detection (3 bucket short/medium/long) hay CSS clamp() với viewport units? JS đơn giản hơn và predictable, CSS clamp tự động hơn nhưng khó tune.
  • Verse reference dưới câu hỏi — có cần hiển thị TRƯỚC khi trả lời không, hay chỉ sau khi sai (per SPEC 4.5)? Hiện screenshot cũ KHÔNG có verse → cần confirm intent.
  • HUD pills (Năng lượng + Combo + Điểm) consolidated thành 1 row — có cần điểm session hiện tại visible không, hay chỉ năng lượng + combo đủ rồi?
  • Có muốn tôi vẽ thêm state đáp án sai → slide-down explanation panel per SPEC 4.5 không?
