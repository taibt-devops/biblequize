Cấu trúc 4 sections (theo thứ tự thông tin):

  • Page header — title + ngày + chip mùa Ngũ Tuần (×1.5 bonus per spec) + countdown làm mới (UTC).
  • Hero card 2 cột — state-aware. State A focus vào CTA + reward preview + yesterday recap. State B focus vào score circle + insight + share.
  • Leaderboard hôm nay với tabs Toàn cầu / Nhóm của tôi — luôn hiện row của user (highlight gold), kể cả chưa làm thì hiện "—" để biết rõ vị trí.
  • Streak + Verse stack — streak với 7-day visualization + freeze count. Verse từ BTTHĐ 2011, có thể luân chuyển hàng ngày.
  • Heatmap 30 ngày — replace 7-day dots cũ. Hover để xem score per ngày, today highlighted. Cho user thấy progress dài hạn.

Điểm fix so với design cũ:

  • Loại bỏ title duplicate (chỉ còn 1 ở header).
  • Hero card không còn rỗng — có status badge animated, reward preview cụ thể, yesterday recap, illustration.
  • Leaderboard có rank + medal + score + time + tier; row của user highlighted; cả 2 state đều show user position.
  • Streak section show 7-day với done/today/freeze, có freeze counter.
  • Câu Kinh Thánh thật từ BTTHĐ 2011 — không còn placeholder rỗng.
  • Thêm chip "Mùa Ngũ Tuần" vì hôm nay 04/05/2026 thuộc Mùa Ngũ Tuần — match seasonal system trong memory.

API gap (cần thêm khi implement):

  • GET /api/daily-challenge/today-leaderboard?scope=global|group&limit=10
  • GET /api/me/daily-challenge/history?days=30 — heatmap data
  • GET /api/me/daily-challenge/yesterday-summary — yesterday recap
  • Backend đã có GET /api/daily-challenge/result với betterThanPercent — reuse cho state B.

Open questions cho Bui:

  • Yesterday recap có giá trị không, hay chỉ noise? Có thể conditional hide nếu user không làm hôm qua.
  • "Luyện lại câu sai (Practice mode)" ở state B — có cần build flow này ngay v1 hay defer? (Practice retry-mode đã có cho regular session, mở rộng cho daily là 1 thay đổi nhỏ.)
  • Heatmap 30 ngày hay 90 ngày? 30 vừa đẹp grid 15×2 nhưng 90 cho long-term motivation tốt hơn (commit 3 tháng = 1 mùa).
